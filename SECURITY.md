# Security Policy

## Supported Versions

| Version | Supported |
|---|---|
| 1.x | Yes |

## Reporting Vulnerabilities

Please do not open a public issue for a vulnerability in Thor itself. Email the maintainer or use GitHub private vulnerability reporting if enabled.

Include:

- Affected version or commit.
- Reproduction steps.
- Impact.
- Suggested fix, if available.

## Responsible Use

Thor is intended only for educational purposes and authorized security assessments. Do not use Thor against systems without permission.

## Out of Scope

Requests to add stealth, exploitation beyond tool-orchestrated authorized testing, credential theft, destructive behavior, persistence, or bypass techniques are out of scope.
